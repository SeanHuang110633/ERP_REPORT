Q2 流程圖總覽

以流程圖方式呈現各功能對應的方法呼叫鏈路

1) 啟動程式到載入 ALV1

從程式入口到畫面 0100 完成三個 ALV 的首次顯示。

  1. START-OF-SELECTION
  2. lcl_app->start()
  3. lcl_app->load_documents()
  4. lcl_app->calculate_document_totals()
  5. CALL SCREEN 0100
  6. PBO: status_0100 / init_0100
  7. lcl_app->init_screen()
  8. lcl_app->display_alvs()

2) ALV1 點擊 -> ALV2

點擊傳票列後載入分錄明細。

  1. ALV1 double click / hotspot
  2. lcl_event_handler->on_doc_*
  3. lcl_app->handle_doc_row()
  4. lcl_app->load_items()
  5. lcl_app->refresh_item_alv()
  6. lcl_app->refresh_check_alv()

3) ALV2 點科目代碼 -> ALV3（科目檢核）

點擊 HKONT 後載入科目主檔檢核。

  1. ALV2 double click / hotspot HKONT
  2. lcl_event_handler->on_item_*
  3. lcl_app->handle_item_row()
  4. lcl_app->load_account_checks()
  5. lcl_app->refresh_check_alv()

4) ALV2 點客戶代碼 -> ALV3（客戶檢核）

點擊 KUNNR 後載入客戶主檔檢核。

  1. ALV2 double click / hotspot KUNNR
  2. lcl_event_handler->on_item_*
  3. lcl_app->handle_item_row()
  4. lcl_app->load_customer_checks()
  5. lcl_app->refresh_check_alv()

5) Risk Only（RISK）

只顯示風險傳票並清空下方明細。

  1. PAI: user_command_0100
  2. lcl_app->handle_user_command('RISK')
  3. lcl_app->filter_risk_only()
  4. lcl_app->refresh_doc_alv()
  5. lcl_app->clear_items_and_checks()
  6. lcl_app->refresh_item_alv()
  7. lcl_app->refresh_check_alv()

6) Show All（ALL）

恢復顯示全部傳票並清空下方明細。

  1. PAI: user_command_0100
  2. lcl_app->handle_user_command('ALL')
  3. lcl_app->show_all_documents()
  4. lcl_app->refresh_doc_alv()
  5. lcl_app->clear_items_and_checks()
  6. lcl_app->refresh_item_alv()
  7. lcl_app->refresh_check_alv()

7) Summary（SUMM）

顯示 OK/WARNING/ERROR 統計。

  1. PAI: user_command_0100
  2. lcl_app->handle_user_command('SUMM')
  3. lcl_app->show_risk_summary()

8) Back / Exit

離開畫面或結束程式。

  1. PAI: user_command_0100
  2. lcl_app->handle_user_command('BACK'/'CANC')
  3. LEAVE TO SCREEN 0
  4. lcl_app->handle_user_command('EXIT')
  5. LEAVE PROGRAM
